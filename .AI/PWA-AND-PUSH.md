### Icon Generator Algorithm (`generate-icons.js`)

The script implements a complete PNG encoder and vector-style renderer from scratch:

**1. PNG Encoder (`makePng`)**:
- Builds PNG file manually: 8-byte PNG signature → IHDR → IDAT → IEND chunks
- IHDR: 8-bit RGBA color type (type 6), width × height
- IDAT: raw pixel rows (filter byte `0x00` per row) compressed with `zlib.deflateSync({ level: 9 })`
- Each chunk: 4-byte length + 4-byte type + data + 4-byte CRC32
- CRC32 computed via lookup table (standard polynomial `0xEDB88320`)

**2. Geometry Layers** (rendered back-to-front):
| Layer | Geometry | Color |
|-------|----------|-------|
| Clip | Rounded rect, corner radius = 22% of size | Transparent outside |
| Background | Full rounded rect | `#0d0d1f` |
| Outer ring | Annulus at 33–43% of half-size radius | `#6366f1` |
| Inner ring | Annulus at 23.5–27.5% of half-size radius | `#a5b4fc` |
| Center dot | Circle at ≤12% of half-size radius | `#c7d2fe` |

**3. Anti-aliasing**: 3×3 supersampling (9 sub-pixel samples per pixel), averaged for smooth edges.

**4. Rounded-Rect Clipping** (`inRoundedRect`): Checks if a point is inside a rectangle with rounded corners using distance-from-corner calculation.

**5. Output**: Writes 3 PNG files to `frontend/` directory:
| Target | Size | Filename |
|--------|------|----------|
| Android home | 512×512 | `icon-512.png` |
| Android splash | 192×192 | `icon-192.png` |
| iOS home | 180×180 | `apple-touch-icon.png` |

## Service Worker (`service-worker.js`)

### Cache Strategy

**Cache name**: `flatmate-portal-v5` (bumped on each release)

**Pre-cached assets** (installed on SW `install` event):
```javascript
const ASSETS = [
  '/Flatmate-Portal/',
  '/Flatmate-Portal/index.html',
  '/Flatmate-Portal/app.js',
  '/Flatmate-Portal/style.css',
  '/Flatmate-Portal/manifest.json',
];
```

**Strategy**: Cache-first for same-origin GET requests, passthrough for cross-origin (API calls)

```
Fetch request
  │
  ├── Non-GET or cross-origin? → Skip (no caching)
  └── Same-origin GET → Check cache
        ├── Cache hit → Return cached response
        └── Cache miss → Fetch from network
```

### Lifecycle Events

1. **Install**: Pre-caches shell assets. Does NOT call `skipWaiting()` — waits for user approval
2. **Activate**: Clears all old caches (any cache name ≠ current `CACHE_NAME`). Calls `self.clients.claim()`
3. **Message**: Listens for `'SKIP_WAITING'` message to activate pending update

### Update Flow

1. App calls `reg.update()` on load to check for SW changes
2. If a new SW is found and installed (waiting state), the yellow "⬆ Update" button appears
3. User clicks Update → sends `'SKIP_WAITING'` to waiting SW
4. SW activates → `controllerchange` event fires → page reloads
5. Fallback: if no waiting SW, unregisters SW, clears all caches, and hard reloads

### Push Event Handler

```javascript
self.addEventListener('push', (event) => {
  let data = event.data ? event.data.json() : {};

  const title = data.title || 'Flatmate Portal';
  const options = {
    body: data.body || '',
    icon: '/Flatmate-Portal/icon-192.png',
    badge: '/Flatmate-Portal/icon-192.png',
    vibrate: data.isEmergency ? [200, 100, 200, 100, 200] : [100],
    requireInteraction: !!data.isEmergency,
    data,
  };

  event.waitUntil(self.registration.showNotification(title, options));
});
```

**Emergency notifications:**
- Triple vibration pattern: `[200, 100, 200, 100, 200]`
- `requireInteraction: true` — notification stays until dismissed

**Normal notifications:**
- Single vibration: `[100]`
- Auto-dismisses

### Notification Click

Clicking a notification opens/focuses the app:
```javascript
self.addEventListener('notificationclick', (event) => {
  event.notification.close();
  // Try to focus existing window, else open new one
  clients.matchAll({ type: 'window', includeUncontrolled: true }).then(...)
});
```

## Web Push Implementation (Worker-side)

The Worker implements Web Push from scratch — no libraries. This is the most complex part of the codebase.

### VAPID JWT Generation (RFC 8292)

Creates an ES256 (ECDSA P-256 + SHA-256) signed JWT for VAPID authentication.

```javascript
async function createVapidJwt(endpoint, env) {
  // 1. Build JWT header: { typ: "JWT", alg: "ES256" }
  // 2. Build JWT payload: { aud: "<endpoint-origin>", exp: now+12h, sub: VAPID_SUBJECT }
  // 3. Base64url-encode both, concatenate with "."
  // 4. Import VAPID private key as JWK (EC P-256)
  // 5. Sign with ECDSA SHA-256
  // 6. Return: header.payload.signature
}
```

**Key details:**
- Audience (`aud`) is the push service origin (e.g., `https://fcm.googleapis.com`)
- Token expires after 12 hours
- Uses Web Crypto API's `crypto.subtle.importKey` and `crypto.subtle.sign`

### Payload Encryption (RFC 8291 — aes128gcm)

Encrypts push notification payloads end-to-end between Worker and browser.

```javascript
async function encryptPayload(plaintextStr, p256dhB64, authB64) {
  // 1. Generate ephemeral ECDH key pair (P-256)
  // 2. Import client's p256dh public key
  // 3. Derive shared secret via ECDH (256 bits)
  // 4. HKDF: derive IKM using auth secret + "WebPush: info" info
  // 5. Generate random 16-byte salt
  // 6. HKDF: derive 128-bit CEK (Content Encryption Key) from IKM
  // 7. HKDF: derive 96-bit nonce from IKM
  // 8. Pad plaintext: content || 0x02 (last record delimiter)
  // 9. AES-128-GCM encrypt
  // 10. Build aes128gcm body:
  //     salt(16) || record_size(4) || idlen(1) || keyid(65) || ciphertext
  // Return: Uint8Array body
}
```

**Crypto operations used:**
- `crypto.subtle.generateKey` — ephemeral ECDH P-256
- `crypto.subtle.deriveBits` — ECDH shared secret + HKDF
- `crypto.subtle.encrypt` — AES-128-GCM

### Sending Push Notifications

```javascript
async function sendPush(subscriptionJSON, payload, env) {
  // 1. Parse subscription JSON
  // 2. Create VAPID JWT for the endpoint
  // 3. Encrypt payload using p256dh + auth keys
  // 4. POST to subscription endpoint with headers:
  //    - Authorization: vapid t=<jwt>, k=<public-key>
  //    - Content-Encoding: aes128gcm
  //    - Content-Type: application/octet-stream
  //    - TTL: 86400 (24 hours)
  //    - Urgency: high (emergency) or normal
  // 5. On 404/410 response: delete expired subscription from DB
}
```

### Broadcast vs Targeted

```javascript
async function broadcastPush(db, payload, env) {
  // SELECT all subscriptions → sendPush() to each via Promise.allSettled()
}

async function targetedPush(db, username, payload, env) {
  // SELECT subscription WHERE username → sendPush() to one
}
```

### Push Notification Payloads

All payloads follow this format:
```json
{
  "title": "📋 New Item Added",
  "body": "Amir added: Buy milk",
  "isEmergency": false
}
```

**Notification triggers:**

| Event                | Title Format              | Target     | Urgency |
|----------------------|---------------------------|------------|---------|
| Item added           | 📋/🚨 New Item Added     | All        | Normal/High |
| Item updated         | ✏️ Item Updated           | All        | Normal   |
| Announcement         | 📢 {username}             | All        | Normal   |
| Trash reminder (cron)| 🗑️ Trash Reminder        | One user   | Normal   |
| Test notification    | 🔔 Test Notification      | All        | Normal   |
